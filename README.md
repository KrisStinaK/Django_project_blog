## Особенности
- Посты с поддержкой тегов и категорий
- Комментарии к постам
- Поиск по постам
- Админка Django для управления контентом
- Поддержка статики и медиа
- Простая настройка локального окружения

## Требования
- Python >= 3.11
- Django >= 3.x
- PostgreSQL
- зависимости в файле requirements.txt

## Установка

1. Клонируйте репозиторий
   - git clone https://github.com/KrisStinaK/Django_project_blog.git
   - cd Django_project_blog

2. Установите зависимости
   - Python виртуальное окружение (рекомендовано)
     - python -m venv venv
     - source venv/bin/activate  # Linux/macOS
     - venv\Scripts\activate     # Windows
   - Установка зависимостей
     - pip install -r requirements.txt

3. Создайте локальную базу данных (пример для PostgreSQL, можно заменить на SQLite)
   - Запустите PGAdmin сервер
   - Создайте пустую бд
   - В файле settings.py найдите следующие настройки и вставте свои данные бд
~~~
   DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'имя таблицы',
        'USER': 'имя пользователя',
        'PASSWORD': 'ваш пароль от сервера',
        'PORT': 'ваш порт',
        'HOST': 'localhost',
    }
}
~~~

4. Миграции
   - python manage.py makemigrations
   - python manage.py migrate

## Конфигурация базы данных
- По умолчанию используется SQLite для локальной разработки.
- Для продакшена рекомендуется PostgreSQL или MySQL.
- Убедитесь, что в settings.py/ settings_local.py корректно настроены DATABASES и статические файлы.

## Запуск проекта
- Запустите сервер
  - python manage.py runserver 8000
- Доступ: http://127.0.0.1:8000/

- Админка доступна по http://127.0.0.1:8000/admin/ (логин и пароль задаются при создании суперпользователя)

Создайте суперпользователя:
- python manage.py createsuperus
